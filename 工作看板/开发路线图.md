# 开发路线图与技术实现指南

## 技术选型理由

### 为什么选择 Tauri？

| 对比项 | Tauri | Electron |
|--------|-------|----------|
| 打包体积 | ~3-10MB | ~60-120MB |
| 内存占用 | 50-150MB | 200-500MB |
| 安全性 | Rust后端，内存安全 | Node.js相对较弱 |
| 构建速度 | 快 | 较慢 |
| 开发门槛 | 需了解Rust | 主要是Web技术 |

**推荐理由**：对于轻量级工具，Tauri在性能和体量上都优于Electron。

#### 3.2 全局状态管理

```typescript
// src/store/task.ts
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import type { Task } from '@/types/task'

export const useTaskStore = defineStore('task', () => {
  const tasks = ref<Task[]>([])
  const loading = ref(false)

  const todoTasks = computed(() =>
    tasks.value.filter(t => t.status === 'todo')
  )

  const inProgressTasks = computed(() =>
    tasks.value.filter(t => t.status === 'in_progress')
  )

  const completedTasks = computed(() =>
    tasks.value.filter(t => t.status === 'done')
  )

  async function fetchTasks() {
    loading.value = true
    try {
      const response = await invoke('get_tasks')
      tasks.value = response as Task[]
    } finally {
      loading.value = false
    }
  }

  async function createTask(task: Partial<Task>) {
    const newTask = await invoke('create_task', { task })
    tasks.value.push(newTask as Task)
    return newTask
  }

  async function updateTask(id: string, updates: Partial<Task>) {
    const updated = await invoke('update_task', { id, updates })
    const index = tasks.value.findIndex(t => t.id === id)
    if (index !== -1) {
      tasks.value[index] = updated as Task
    }
  }

  async function deleteTask(id: string) {
    await invoke('delete_task', { id })
    tasks.value = tasks.value.filter(t => t.id !== id)
  }

  return {
    tasks,
    loading,
    todoTasks,
    inProgressTasks,
    completedTasks,
    fetchTasks,
    createTask,
    updateTask,
    deleteTask
  }
})
```

#### 3.3 Tauri 后端命令

```rust
// src-tauri/src/commands.rs
use tauri::State;
use crate::db::Database;

#[tauri::command]
pub async fn get_tasks(db: State<'_, Database>) -> Result<Vec<Task>, String> {
    db.get_all_tasks()
}

#[tauri::command]
pub async fn create_task(
    db: State<'_, Database>,
    task: TaskInput
) -> Result<Task, String> {
    db.create_task(task)
}

#[tauri::command]
pub async fn update_task(
    db: State<'_, Database>,
    id: String,
    updates: TaskInput
) -> Result<Task, String> {
    db.update_task(id, updates)
}

#[tauri::command]
pub async fn delete_task(
    db: State<'_, Database>,
    id: String
) -> Result<(), String> {
    db.delete_task(id)
}

// 在 main.rs 中注册
fn main() {
    tauri::Builder::default()
        .setup(|app| {
            let db = Database::new()?;
            app.manage(db);
            Ok(())
        })
        .invoke_handler(tauri::generate_handler![
            get_tasks,
            create_task,
            update_task,
            delete_task,
            // 其他命令...
        ])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

#### 4.3 AI工作总结模块（Day 7-9）

**AI服务实现**：
```typescript
// src/services/ai-service.ts
import { invoke } from '@tauri-apps/api/tauri'

export interface AISummaryRequest {
  tasks: any[]
  period: { start: Date; end: Date }
  context?: string
}

export interface AISummaryResponse {
  summary: string
  achievements: string[]
  challenges: string[]
  suggestions: string[]
}

export class AIService {
  private static instance: AIService
  private config: any = null

  private constructor() {}

  static getInstance(): AIService {
    if (!AIService.instance) {
      AIService.instance = new AIService()
    }
    return AIService.instance
  }

  async getConfig(): Promise<any> {
    if (!this.config) {
      this.config = await invoke('get_settings')
    }
    return this.config.aiConfig
  }

  async generateSummary(request: AISummaryRequest): Promise<AISummaryResponse> {
    const config = await this.getConfig()

    if (!config.apiUrl) {
      throw new Error('AI API 地址未配置')
    }

    const prompt = this.buildSummaryPrompt(request)

    try {
      const response = await fetch(`${config.apiUrl}/chat/completions`, {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${config.apiToken || ''}`
        },
        body: JSON.stringify({
          model: config.model || 'gpt-3.5-turbo',
          messages: [
            {
              role: 'system',
              content: '你是一个专业的工作助手，擅长分析工作内容并生成清晰、结构化的工作总结。'
            },
            {
              role: 'user',
              content: prompt
            }
          ],
          temperature: 0.7
        })
      })

      if (!response.ok) {
        throw new Error(`API请求失败: ${response.statusText}`)
      }

      const data = await response.json()
      return this.parseAIResponse(data)
    } catch (error) {
      throw new Error(`AI总结生成失败: ${error}`)
    }
  }

  private buildSummaryPrompt(request: AISummaryRequest): string {
    const tasksByStatus = {
      completed: request.tasks.filter(t => t.status === 'done'),
      inProgress: request.tasks.filter(t => t.status === 'in_progress'),
      pending: request.tasks.filter(t => t.status === 'todo')
    }

    const tasksSummary = `
完成的工作 ${tasksByStatus.completed.length}项：
${tasksByStatus.completed.map(t => `- ${t.title}: ${t.description}`).join('\n')}

进行中的工作 ${tasksByStatus.inProgress.length}项：
${tasksByStatus.inProgress.map(t => `- ${t.title}: ${t.description}`).join('\n')}

待处理的工作 ${tasksByStatus.pending.length}项：
${tasksByStatus.pending.map(t => `- ${t.title}: ${t.description}`).join('\n')}
`

    return `
请根据以下工作内容生成一份周工作总结报告：

时间范围：${request.period.start.toLocaleDateString('zh-CN')} ~ ${request.period.end.toLocaleDateString('zh-CN')}

${tasksSummary}

请按照以下格式提供总结：
1. 本周工作概述（2-3句话）
2. 主要工作成果（分条列出）
3. 遇到的挑战和问题
4. 下周工作建议和计划

请用专业、简洁的语言，突出工作重点。
`
  }

  private parseAIResponse(data: any): AISummaryResponse {
    const content = data.choices?.[0]?.message?.content || ''

    // 这里可以添加更复杂的解析逻辑
    // 目前简单返回AI返回的原始内容
    return {
      summary: content,
      achievements: [],
      challenges: [],
      suggestions: []
    }
  }
}
```

## 常见问题解决

### Q1: Tauri 开发依赖安装失败

```bash
# Windows 需要 Visual Studio C++ 构建工具
# 下载安装：https://visualstudio.microsoft.com/visual-cpp-build-tools/

# 然后重试
npm install
```

### Q2: 数据库迁移失败

```bash
# 删除现有数据库和迁移
rm -rf prisma/migrations
rm workboard.db

# 重新生成
npx prisma migrate dev --name init
```

### Q3: 构建时出现证书错误

```bash
# macOS 需要签名证书
# 跳过签名（仅开发测试）
npm run tauri build -- --no-bundle
```

## 性能优化建议

1. **虚拟滚动**：任务列表使用虚拟滚动
2. **懒加载**：路由组件和大型图表库懒加载
3. **防抖节流**：搜索和自动保存使用防抖节流
4. **缓存策略**：使用 Pinia 持久化插件缓存常用数据
5. **数据库索引**：为常用查询字段添加索引

## 总结

通过这个详细的开发路线图，你可以按步实现整个工作看板应用。关键点：

1. 先完成基础架构，确保数据流畅通
2. 核心功能优先，逐步添加高级功能
3. 预留扩展接口，便于后续迭代
4. 注意性能和用户体验的平衡
